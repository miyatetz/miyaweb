# 記事の書き方
## 記事を作る
マークダウン記法で書くことで記事を追加出来ます。

まずは.md形式のファイルを`_drafts`フォルダ内に作成してください。

1. 以下の赤枠部分の`_drafts`をクリックします。
![図1](https://github.com/user-attachments/assets/8170ac95-6f3e-41dd-bf9e-0c14a47cdde9)
---
2. 右上の`Add file` → `Create new file` をクリックします。
![CreateNewFile](https://github.com/user-attachments/assets/50f6c667-3980-490f-b6af-8faa6d4b3bd0)
---
3. 上部の`Name your file…` につけたいファイル名をつけてください。
その際、名前は「`2025-04-20-titlename.md`」のようにすると整理しやすいです。
![fileName](https://github.com/user-attachments/assets/777b3f76-1297-474e-a058-1b67332d11c1)
---
4. 次に、ファイルの初めに以下の内容を記述します。(コピペできます）
```markdown
---
layout: post
title:  "記事のタイトル"
date:   YYYY-MM-DD 22:00:00 +0900
categories: blog
---
```
※`categories` の部分は、基本的には`blog` のままでよいです。

※`YYYY-MM-DD` の部分は具体的には`2026-04-24` のように、作成した年月日を書いてください。
未来の日時を指定した場合、その時間までは記事は非公開となります。

※`date` の部分の後半の`+0900` については、日本時間であることを示しているため、特別な理由がなければ変更しないでください。


## 記事を書く
先ほど記述した内容の次の行から本文を書くことができます。

マークダウンでは、段落や太字, 画像の挿入など、様々なことが可能です。

Markdownの主な書き方（詳しくは[こちら](https://gist.github.com/mignonstyle/083c9e1651d7734f84c99b8cf49d57fa)）

| 記法 | 書き方 | 注意点 |
| --- | --- | --- |
| 見出し | `# 大見出し`<br>`## 中見出し` | `#` の後に半角スペースが必要 |
| 太字 | `**太字**` | `**` の前後にスペースは不要 |
| 箇条書き | `- 項目1` | `-` の後に半角スペースが必要 |
| リンク | `[表示文字](URL)`  | 記号はすべて半角。<br>間にスペースは不要。 |
| 画像 | `![説明](URL/ファイルパス)`  | `説明`は画像が出ないときの代替文字。<br>自分で用意した画像の挿入方法は後述 |

※記号（`#` `*` `-` `[] ()` !）はすべて半角で書いてください。

編集画面左上の`Edit` の横の`Preview` をクリックすると、実際の記事の見た目が確認できます。

**記事を書き終えたら**、編集画面右上の`Commit Changes…` をクリックし、`Commit message` の入力欄に適当なタイトルをつけ、右下の`Commit changes` をクリックすると、記事が保存されます。
（この時点では、まだ記事は公開されていません。）

![commitChanges](https://github.com/user-attachments/assets/2b75ede7-2b8f-44ee-a907-fc65ff6abba9)


### 画像の追加について
画像などを追加する場合は`assets`フォルダ内に追加(アップロード)してそのパスを指定してください。

```
![キャプション]({{site.baseurl}}/assets/2026/sample.png)
```

上の例は、パスが`assets/2026/sample.png`である場合の例です。

`{{site.baseurl}}/`の後にパスを指定する感じです。

パスについては画像ファイルの位置を開き、右上の三点マークをクリックして`copy path`を選択するとコピーできます。

## 記事を公開する
記事を公開するには公開したい記事のマークダウンファイルを`_drafts`フォルダから`_posts`フォルダに移動してください。

1. 編集モードでないことを確認してください。

2. `.`（ピリオド）キーを押すと次のような画面が出てきます。（少し時間がかかります）

![vscodeweb](https://github.com/user-attachments/assets/e34dfc0a-4346-4d29-b9ce-575099c9ebca)

3. 上の四角の部分（`_drafts`フォルダ内）の作成記事を、
下の四角の部分（`_posts`フォルダ内）にドラック＆ドロップで移動させてください。

4. 正しく移動したことを確認して、記事の投稿が完了します。
記事の公開のときに開いたタブは削除してかまいません。

# 保守の仕方
## ホームページの内容を変更する
ホームページの内容を変更する場合は`index.markdown`の内容を変更してください。

## `about`の内容を変更する
`about`内の内容を変更したい場合は`about.markdown`の内容を変更してください。

## サイトが更新されないときは
このサイトはGitHub PagesのActionsにあるJekyllを用いてマークダウンからhtmlやcssをコミットされるたびに生成しています。

そのため記事公開のコミットをしても、生成が終わるまでは更新されません。

もし、３分以上待っても更新されなかった場合、何かしらの問題が発生したと考えられます。

### GitHub Actionsの確認
このレポジトリの`Actions`タブから正しく生成されているかを確認できます。最新のコミットに対しての生成が失敗している場合ここに表示されます。

### Jekyll on Windowsでの確認
GitHub上でちゃんと動かない場合、JekyllをWindowsで動作させることで問題を特定できる場合があります。

このリポジトリを自分の端末にクローンして試してください。
